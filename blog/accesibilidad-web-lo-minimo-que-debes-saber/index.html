Accesibilidad web: lo mínimo que debes saber

La accesibilidad web es la práctica de construir sitios y aplicaciones que puedan ser usados por todas las personas, independientemente de sus capacidades físicas, cognitivas o tecnológicas. No es un extra opcional: en muchos países es un requisito legal, y en todos es una responsabilidad ética.

HTML semántico como base

La accesibilidad empieza por escribir HTML correcto. Los elementos semánticos no solo estructuran el documento visualmente; también comunican su significado a tecnologías asistivas como los lectores de pantalla. Algunas reglas fundamentales:

  • Usa el elemento correcto para cada propósito: <button> para acciones, <a> para navegación, <h1>–<h6> para jerarquía de títulos.
  • No uses <div> ni <span> para simular botones o enlaces si puedes usar el elemento nativo.
  • Siempre incluye el atributo lang en la etiqueta <html> para indicar el idioma del contenido.
  • Las imágenes deben tener siempre un atributo alt descriptivo; si la imagen es decorativa, usa alt="".

Contraste y legibilidad

Aproximadamente el 8% de los hombres tienen algún grado de daltonismo. Las pautas WCAG establecen que el contraste entre texto y fondo debe ser de al menos 4.5:1 para texto normal y 3:1 para texto grande. Para verificar el contraste de tus colores puedes usar herramientas gratuitas en línea como WebAIM Contrast Checker.

Además del contraste de color, la legibilidad también depende del tamaño de la fuente. Nunca uses texto menor a 14px en contenido principal, y asegúrate de que el interlineado sea suficientemente amplio para facilitar la lectura.

Navegación accesible

Muchas personas navegan la web únicamente con el teclado, sin usar el ratón. Para garantizar que tu sitio sea navegable en teclado:

  • Todos los elementos interactivos (enlaces, botones, formularios) deben ser alcanzables con la tecla Tab.
  • El indicador de foco (el borde que aparece al navegar con Tab) nunca debe eliminarse con outline: none sin reemplazarlo por un estilo alternativo visible.
  • El orden de los elementos en el DOM debe seguir el mismo orden lógico que el visual.
  • Proporciona un enlace de «saltar al contenido» al inicio de la página para que los usuarios de teclado puedan omitir el menú de navegación.

Buenas prácticas esenciales

Estas prácticas cubren la mayoría de los casos de accesibilidad con un esfuerzo mínimo:

  • Etiqueta todos los campos de formulario con <label>; nunca uses solo el atributo placeholder como sustituto.
  • Usa aria-label o aria-describedby cuando el contexto visual no es suficiente para un lector de pantalla.
  • Las tablas de datos deben tener encabezados con <th> y el atributo scope correspondiente.
  • Valida tu página con herramientas automáticas como axe o Lighthouse, pero recuerda que ninguna herramienta detecta todos los problemas de accesibilidad.

Construir accesible no es más difícil que construir sin accesibilidad. Es cuestión de hábito. Empieza por las bases y verás que la mayoría de las mejoras no requieren esfuerzo adicional, solo conciencia.
